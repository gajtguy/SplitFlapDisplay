# Split-Flap Display

> **Note:** This file was generated by opencode and may contain errors. Please verify all information before proceeding.

A modular split-flap display system for displaying characters, controllable via RS485 from a Raspberry Pi.

**YouTube Video:** https://www.youtube.com/watch?v=-C8_AtxEEQc

**Created by:** Adam G Makes

**YouTube Channel:** https://www.youtube.com/@AdamGMakes

---

## Project Overview

This is aDIY split-flap display project featuring:

- 64 characters per module (A-Z, 0-9, and special characters)
- Modular design - chain multiple modules together
- RS485 communication for reliable long-distance control
- Raspberry Pi frontend with web interface
- 3D printed parts using Bambu Lab or similar printer

---

## Hardware

### Electronics

| Module | Description |
|--------|-------------|
| SplitFlapDriverATtiny1616 V2 | Custom PCB with ATtiny1616 microcontroller, ULN2003 stepper driver, SN65HVD72 RS485 transceiver, SPX3819 3.3V regulator |
| SplitFlapBusBoard | Shared bus board connecting multiple modules |

### CAD Files

All CAD files are located in the `CAD/` folder:
- Character Drum & Lip - Holds each flap
- Center Gear - Main gear driving the flipper mechanism
- Motor Gear - Connects to stepper shaft
- Gear Plate - Mounting plate for motor and gears
- Module Enclosure - Housing for the module
- DIN Rail Mount - Optional mounting solution
- 64 unique Flap files - One for each character

## Firmware

Located in `SplitFlapFirmware/splitflapfirmwarev6/`:

- Arduino-based firmware for ATtiny1616
- RS485 half-duplex communication at 115200 baud
- EEPROM-based calibration storage
- Hall effect sensor homing
- Supports 64 unique characters

## Assembly Instructions

1. **Print Parts** - Use the provided Bambu Lab print profile in `CAD/64FlapsWithLetters/`
2. **Assemble Mechanics** - Follow the video tutorial for gear and drum assembly
3. **Program PCB** - Flash the firmware via UPDI programmer
4. **Calibrate** - Home each module and store positions in EEPROM
5. **Connect Bus** - Chain modules via the Bus Board
